# Politique de sécurité

## Versions prises en charge

Les correctifs de sécurité sont appliqués en priorité à la dernière version stable publiée d’Atlas SI.

| Version | Prise en charge |
|---|---|
| 2.2.x | Oui |
| versions antérieures | Non garantie |

## Signaler une vulnérabilité

Merci de ne pas publier immédiatement une vulnérabilité exploitable dans un ticket public.

Incluez si possible :

- la version concernée ;
- les conditions nécessaires à l’exploitation ;
- les étapes de reproduction ;
- l’impact estimé ;
- les journaux ou captures utiles, sans donnée sensible ;
- une proposition de correctif, le cas échéant.

Un accusé de réception sera envoyé dès que possible. La vulnérabilité pourra être publiée de manière coordonnée après mise à disposition d’un correctif.

## Données sensibles

Atlas SI peut contenir une représentation détaillée d’un système d’information. Les utilisateurs doivent protéger les sauvegardes, exports, fichiers de configuration, certificats et données de démonstration comme des informations potentiellement sensibles.
